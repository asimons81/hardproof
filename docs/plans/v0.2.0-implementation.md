# v0.2.0 Gatehouse Implementation Plan

Status: Implemented; v0.2.0 release candidate prepared for independent audit.

## Task 0: Freeze v0.2 contracts

- Add contract tests for version metadata, command registration, stable v0.1 rule keys, trace schema, and migration version.
- Add ADR 0006 for ordered policy precedence, immutable-rule ownership, and caller-supplied clock determinism.
- Gate: contracts fail for only the missing v0.2 behavior.

## Task 1: Add policy domain model and migration 002

- Add immutable trace, rule, waiver, risk suggestion, and persisted decision models.
- Add migration 002 and repository methods with append-only decision traces and waiver lifecycle events.
- Test fresh v2, v1 upgrade, rollback, idempotence, data preservation, constraints, and concurrent reads.
- Gate: all migration and model tests pass; no v1 schema mutation.

## Task 2: Implement deterministic terminal normalization

- Add bounded tokenizer and chain classifier for POSIX/PowerShell forms and common wrappers.
- Preserve every v0.1 stable rule key and add documented category keys.
- Test quoting, chaining, malformed input, platform paths, encoded/secret-like values, and immutable commands.
- Gate: classifier property tests prove identical inputs produce identical normalized output and trace.

## Task 3: Implement strict project policy configuration

- Evolve config schema to v2 with named allow/deny/approval rules and failure mode.
- Add size limits, regex compilation, path/stage/profile validation, and actionable diagnostics.
- Add config v1-to-v2 migration guidance without silently rewriting user files.
- Gate: valid configs round-trip; unknown, weakening, ambiguous, and secret-expanding configs fail closed.

## Task 4: Build the single traceable policy engine

- Refactor the existing evaluator into ordered rules returning full traces and fingerprints.
- Integrate immutable, project, stage, pack, and default rules without parallel evaluators.
- Persist bounded/redacted decision records through the existing hook path.
- Gate: every final action has one final key and an ordered trace; v0.1 behavior remains compatible.

## Task 5: Add scoped human waiver lifecycle

- Implement create/list/revoke/expire matching services and storage.
- Expose mutation only through human command surfaces; tools remain read-only.
- Prove scope intersection, expiry, revocation, restart survival, audit completeness, and immutable-rule refusal.
- Gate: generated waiver scopes cannot bypass immutable, evidence, approval-authenticity, state, or migration rules.

## Task 6: Add policy explain surfaces

- Add CLI, slash, and safe tool handlers for historical-event and hypothetical-call explanation.
- Provide stable JSON and concise human output with hashes/redacted summaries.
- Gate: allowed, blocked, approval, and waived examples reproduce the durable stored decision.

## Task 7: Add risk suggestions and overrides

- Implement deterministic advisory classifier with reasons and confidence-free categorical output.
- Persist accepted/overridden outcomes and mandatory override rationale.
- Integrate task creation and status output without automatically changing profiles.
- Gate: security, migration, deployment, concurrency, billing, and ordinary-file fixtures classify consistently.

## Task 8: Add bounded stage-graph configuration

- Validate supported optional-stage and additive-gate configuration.
- Compile configuration into the existing forward transition evaluator.
- Test cycle/reorder/VERIFY bypass/core-approval weakening refusals and valid additive gates.
- Gate: defaults exactly preserve v0.1; configured graphs remain finite and monotonic in safety.

## Task 9: Ship four built-in policy packs

- Add Python, Node, Rust, and Go local detection, command categories, risk paths, and check suggestions.
- Support explicit pack selection plus deterministic auto-detection with trace.
- Add mixed-repository precedence tests and package-data contracts.
- Gate: representative fixtures detect without subprocess, network, or global writes.

## Task 10: Improve config, migration, and doctor diagnostics

- Add schema/key/source locations, policy fingerprint, active packs, pending migrations, dry-run migration plan, and recovery guidance.
- Keep diagnostics read-only except explicit migration command.
- Gate: corrupt/newer databases and invalid configs produce safe actionable output without mutation.

## Task 11: Documentation and security review

- Document configuration schema, rule-key catalog, trace format, waiver threat model, packs, stage boundaries, migration, and examples.
- Update README, CHANGELOG, architecture, compatibility, protocol, security, and release docs.
- Run secret, network, placeholder, unsafe-regex, and immutable-waiver reviews.
- Gate: public claims match tested behavior and no security limitation is implicit.

## Task 12: Integration, compatibility, and release candidate

- Run real allow/block/approval/waiver flows through Hermes-compatible hooks and commands across restart.
- Upgrade a v0.1 fixture through v2 and prove reports/evidence remain readable.
- Run full suite, critical coverage, Ruff, mypy, build, Twine, clean wheel, audit, SBOM, checksums, registration, and local-only checks.
- Record platform CI and publication evidence honestly; do not tag while external gates remain blocked.
- Gate: `docs/codex/releases/v0.2.0.md` maps every acceptance criterion to inspectable evidence.

## Release-wide gates

Every task updates `docs/codex/STATUS.md`. Each task begins with a focused failing test, ends with focused and adjacent tests, and receives one coherent commit. Any ambiguity that changes policy authority, persistence, or public behavior requires an ADR. v0.2.0 does not pass until migration, security, compatibility, documentation, package, and release evidence all pass.
