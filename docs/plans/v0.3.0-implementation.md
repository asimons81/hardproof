# v0.3.0 Workcells Implementation Plan

Status: In implementation.

1. Freeze Workcells contracts, public-Hermes compatibility limits, specification, and ADRs.
2. Add migration 003 and durable graph, task, attempt, claim, lifecycle, retry, and escalation records.
3. Implement deterministic graph validation, task/attempt transitions, wave planning, and SQLite claims.
4. Implement bounded artifact paths, brief/context rendering, result contract validation, and redaction.
5. Add the public-Hermes adapter with a deterministic fake and fail-closed recovery.
6. Integrate profiles, stages, policy, configuration, diagnostics, reports, context hooks, six tools, CLI, and slash commands.
7. Add unit, integration, migration, concurrency, recovery, security, contract, and E2E tests.
8. Complete compatibility, security, migration, installation, and operator documentation; then run release gates and request independent audit.

Every behavior step begins with refusal- and success-path tests. Publication,
tagging, and v0.4.0 work remain out of scope until all implementation and
independent-audit gates are complete.
