# Inspiration and Clean-Room Boundary

Crucible is an original, clean-room implementation of a persistent engineering protocol for Hermes Agent.

The Superpowers workflow project provided conceptual inspiration for the broad idea that coding agents benefit from an explicit, inspectable process. That acknowledgement grants no permission to reproduce its implementation or expression. Crucible contributors must not copy or adapt Superpowers source code, skills, prompts, test fixtures, diagrams, templates, repository layout, or documentation wording.

All Crucible behavior, terminology, schemas, prose, tests, and implementation must be derived independently from this project's public requirements and the documented public Hermes Agent APIs. Similar engineering goals must be expressed through original design and independently written code.

Crucible is a working product name pending trademark and marketplace clearance. The project uses the distinct distribution name `crucible-agent`, import package `crucible_agent`, and Hermes plugin key `crucible`. It does not imitate Atlassian branding and is not affiliated with, endorsed by, or sponsored by Atlassian.
