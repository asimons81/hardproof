# Implementation Plan

## Purpose
Turn the approved design into independently testable implementation tasks.

## Required sections
Task order, dependencies, files, acceptance checks, risks, and rollback.

## No-placeholder check
Every task must name observable completion evidence.

## Approval status
Record the human plan gate separately; prose cannot grant approval.

## Unresolved questions
List only questions that do not invalidate the plan.

## Evidence links
Link the approved design and relevant project files.
